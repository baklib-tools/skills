# Baklib 网站复刻流程

基于参考网站，分析设计风格、提取设计元素、适配客户需求，使用 **Baklib 模板系统**（Liquid）实现站点复刻。

**语法与对象规范**见主技能 [baklib-theme-dev](https://github.com/baklib-tools/skills/blob/main/skills/baklib-theme-dev/SKILL.md)（按需安装：`npx skills add baklib-tools/skills --skill baklib-theme-dev`）。

## 确认门（工作流）

收到任务后**先**用简短编号列出拟执行步骤（例如：仅做设计分析 / 是否输出完整 `themes/` 目录 / 是否依赖客户品牌素材）。对**可选或高成本步骤**（大量页面还原、调用付费图床/出图、写入线上环境等）须征求用户明确同意后再执行。

## 使用场景

- 复刻参考网站
- 分析网站设计
- 提取设计元素
- 适配客户需求
- 制作 Baklib 网站模板

## 工作流程概览

1. **网站分析** → 结构、内容、功能分析
2. **设计元素提取** → 色彩、字体、布局、组件
3. **客户需求收集** → 栏目、内容、品牌元素
4. **适配方案制定** → 语言、内容、风格调整
5. **开发计划制定** → 模板结构、组件、优先级
6. **开发实施** → 模板开发、测试
7. **质量检查** → 设计还原度、功能完整性

## 详细流程

### 步骤 1：网站分析

#### 1.1 页面结构分析

- **首页结构**：Hero 区域、特色内容、功能模块、CTA 区域、Footer
- **导航结构**：主导航、二级导航、页脚导航
- **页面类型**：首页、列表页、详情页、表单页

#### 1.2 内容模块分析

- **内容区块**：标题层级、文本内容、图片、按钮
- **功能模块**：搜索、筛选、表单、轮播
- **交互元素**：悬停效果、动画、过渡效果

### 步骤 2：设计元素提取

#### 2.1 色彩系统

使用浏览器开发者工具检查 CSS，提取：

- **主色调**：主色、辅助色、强调色
- **中性色**：背景色、文字色、边框色
- **功能色**：成功、警告、错误、信息

#### 2.2 字体系统

- **字体族**：主字体、备用字体
- **字号层级**：H1、H2、H3、正文、辅助
- **字重和行高**

#### 2.3 布局系统

- **容器宽度**：最大宽度、内边距
- **网格系统**：列数、间距
- **响应式断点**：移动端、平板、桌面

#### 2.4 组件样式

- **按钮**：主要、次要、文字按钮
- **卡片**：阴影、圆角、边框
- **表单**：输入框、选择框

### 步骤 3：客户需求收集

#### 需求清单

- **基本信息**：公司名称、行业、目标用户
- **栏目需求**：需要/不需要的栏目
- **内容偏好**：内容风格、图片风格
- **品牌元素**：Logo、品牌色、品牌字体
- **功能需求**：搜索、表单、特殊功能

### 步骤 4：适配方案

#### 语言适配

- **字体选择**：中文字体（思源黑体、苹方等）
- **排版优化**：中文字符间距、行高调整

#### 内容适配

- **栏目调整**：根据客户需求调整
- **内容规划**：制定内容收集清单
- **品牌替换**：Logo、品牌色替换方案

### 步骤 5：开发计划

#### 模板结构

```
themes/[scope]/[theme_name]/
├── layout/theme.liquid
├── templates/
│   ├── index.liquid
│   ├── list.liquid
│   ├── detail.liquid
│   └── page.liquid
├── snippets/
│   ├── _header.liquid
│   ├── _footer.liquid
│   └── _card.liquid
├── statics/           # 静态页面
└── assets/
```

#### 开发优先级

1. **第一阶段**：基础布局、Header、Footer、样式系统
2. **第二阶段**：首页、列表页、详情页模板
3. **第三阶段**：功能模块（搜索、表单）
4. **第四阶段**：响应式优化、性能优化

### 步骤 6：静态页面

在后台页面创建之前，可使用静态页面提供可访问页面：

- **目录**：`statics/`
- **访问路径**：`/s/[文件名]`
- **示例**：`statics/about-us.liquid` → `/s/about-us`

（路径与约定以 [baklib-theme-dev](https://github.com/baklib-tools/skills/blob/main/skills/baklib-theme-dev/SKILL.md) 中 `references/` 为准。）

### 步骤 7：占位图片

客户未提供图片时，可使用占位图服务（示例，以实际可用服务为准）：

- **Lorem Picsum**：`https://picsum.photos/800/600`

```liquid
{% assign hero_image = page.settings.hero_image | default: 'https://picsum.photos/1920/1080' %}
```

```markdown
### 首页内容
- [ ] Hero 区域标题和描述
- [ ] Hero 背景图（1920x1080px）
- [ ] 特色内容

### 联系信息
- [ ] 公司地址、电话、邮箱
- [ ] 社交媒体链接
```

## 重要规则

1. **充分分析**：在开发前必须充分分析参考网站
2. **需求确认**：必须与客户确认需求后再开始开发
3. **渐进开发**：先实现核心功能，再添加高级特性
4. **质量保证**：确保设计还原度和功能完整性
5. **文档记录**：记录分析结果、开发计划、内容清单

## 质量检查清单

### 设计还原度

- 色彩系统一致
- 字体系统一致
- 布局结构一致
- 组件样式一致

### 功能完整性

- 所有页面正常显示
- 导航功能正常
- 表单功能正常
- 响应式正常

### 内容适配

- 内容已替换为客户内容
- 语言已适配为中文
- 品牌元素已替换
