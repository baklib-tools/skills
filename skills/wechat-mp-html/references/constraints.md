# 微信公众号 HTML：结构与复制规范

## 页面区域划分

| 区域 | 元素 / ID | 说明 |
|------|-----------|------|
| 标题（仅预览） | `h1` 或页面主标题 | 正式发布时在公众号后台填写标题；本地文件可保留便于预览。 |
| 元信息（仅预览） | `#meta_content` | 作者、日期等；后台单独填写。 |
| **正文（唯一需复制）** | **`#js_content`** | 复制该 **`<section>` 的 outerHTML** 到图文正文编辑器；粘贴为富文本。 |

## 标准流程

1. 在 Markdown 或其它稿本中定稿文案与配图需求。
2. 生成本地 HTML，将全部可发布正文放在 **`#js_content`** 内。
3. 浏览器打开文件，检查版式与图片。
4. 点击 **「复制正文到公众号」**，在公众号新建图文正文中 **粘贴**；若剪贴板 API 失败，可在开发者工具中手动复制 `#js_content` 的 HTML。

**说明**：粘贴**不会**带上本页 `<head>` 中的 CSS，因此正文内标题条、色块等必须已写成 **inline style**。

## 正文 DOM 约定（与微信渲染习惯对齐）

以下约定优先保证**粘贴后版式稳定**；不同公众号编辑器版本可能存在细微差异。

### 1. 根节点与块级结构

- 正文根：`<section id="js_content" style="...">`（外层可有额外 `class` 供本地预览，但**正文内部子块不要依赖 class**）。
- **复杂版式**（步骤条、数字标题、多栏）用**多层嵌套 `section`** + inline 样式实现。
- **文字段落**：用 **`<section style="...">`** 包裹文案（**不要使用 `<p>`**），以保持与当前团队模板及编辑器过滤规则一致。
- **不要使用 `<div>`** 作为正文块级容器。

### 2. 标题与列表

- 章节标题可使用 **`h2`～`h4`** 并带 inline style；或采用下文「组件」中的大标题块。
- **列表**：优先用**多个带 `padding-left` 的 `section`** 模拟列表项；若使用 `ul`/`ol`，请在目标编辑器中**实测**粘贴效果后再定稿。

### 3. 行内强调

- 加粗使用 **`<strong style="...">`**。
- **`span`**：部分编辑器会为文字插入带 `leaf` 等属性的 `span`；**手工生成时尽量少用 `span`**。若使用：仅在 `section` 段落内使用；**`span` 内不要再嵌套 `img`/`strong` 等**，以免短行错乱。需强调时优先 `strong`。

### 4. 图片

- `<img style="max-width:100%;height:auto;display:block;margin:20px 0;...">` 等常用写法。
- 正文内图片最终需为**公众号可访问的绝对 URL**（或后台插入生成的链接）。

### 5. 空格与换行

粘贴后，部分编辑器会把空格规范为 `&nbsp;`。编写 HTML 时**避免为排版随意断行或加多余空格**，以免出现多余空白。

## 配图（运营向）

- 长文常见结构：**三图**（痛点 → 方案 → 结果/CTA），篇首/篇中/篇尾各一。
- 图片路径在本地可用相对路径预览；发布前替换为线上 URL。

## 「复制正文到公众号」标准实现

须同时满足：

- 按钮：`id="copy-to-weixin"`（`type="button"`），固定文案如「复制正文到公众号」。
- 提示：`id="copy-toast"`，复制成功/失败时短暂显示。
- 逻辑：读取 `#js_content`，取 **`outerHTML`**（若根即 section）；使用 **`navigator.clipboard.write`** + **`ClipboardItem`**，同时提供 `text/html` 与 `text/plain`；失败时降级为 `execCommand('copy')` 复制同一段 HTML 字符串。

**禁止**：仅使用旧版自定义 class 按钮、`alert()` 提示、或不提供 `text/html` 的纯文本复制作为主路径。

完整可运行示例见同目录 [shell-template.html](shell-template.html)。

## 发布前检查清单

- [ ] 页面存在 **「复制正文到公众号」** 按钮且为上述标准实现。
- [ ] `#js_content` 内正文**不依赖**页面级 `<style>`。
- [ ] 正文块级结构以 **`section` 为主**，**无 `class`** 依赖。
- [ ] 文内图片已替换为**可公开访问**的 URL（或计划在后台插入）。
- [ ] 点击复制后，在公众号编辑器中试粘贴，必要时微调。
