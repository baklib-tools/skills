# 公众号长文：类型结构与组件化（编辑向）

> 目标：用**稳定结构**提升排版效率与品牌一致性；具体 HTML 语法见 [components.md](components.md) 与 [constraints.md](constraints.md)。

## 常见文章类型

### 1. 对比评测 / 选型（软文型）

适用：选型期读者、外站同步软文。

建议结构：

- 标题  
- 导语（场景 + 人群 + 结论预告）  
- 可选：目录  
- 选型结论摘要（3～5 条）  
- 对比维度（按**场景**拆，而非堆功能名）  
- 至少 1～2 个「角色视角」小节  
- 快速验证清单（如何试用 / 如何评估）  
- CTA（试用 / 预约 / 文档）

### 2. 教程 / 操作指南（承接型）

适用：视频延伸阅读、帮助中心同步、产品内引导。

建议结构：

- 标题（「怎么用 / 如何配置 / N 步法」）  
- 本文解决什么问题（一段）  
- 适合谁（角色 / 团队）  
- 前置条件（权限、账号、资料）  
- 操作步骤（**步骤卡片** + 截图）  
- 常见问题 / 注意事项  
- 延伸阅读  
- CTA

### 3. 产品更新汇总（节奏型）

适用：月度/季度更新、活动期传播。

建议结构：

- 标题（时间 + 主题）  
- 本期重点（≤3 条）  
- 按模块列功能更新  
- 影响人群  
- 下期预告  
- CTA

## 组件化思路

将重复出现的块定义为「组件」，在 HTML 中整段复用，例如：

- 导语块、步骤数字标题、总结框、截图说明块、CTA 块、二维码位（若政策允许）。

## 跨渠道一致

若团队同时维护**官网博客**与**公众号**，建议信息结构（标题层级、步骤顺序、关键结论）保持一致；公众号仅在导语长度、配图节奏上做适配。

## 与短视频联动（可选）

教程文可在文首或文末标注对应短视频编号或口令，便于运营串联「短内容 → 长文」。
