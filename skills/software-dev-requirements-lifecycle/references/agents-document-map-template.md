# AGENTS.md 文档地图（最小可粘贴模板）

将以下内容复制到新项目根目录的 `AGENTS.md`，按实际路径改写表格中的文件名；**真源优先级**请与团队共识一致。

---

```markdown
# <项目名> — 协作与文档约定（AGENTS）

面向人类与 AI：**先对齐文档真源，再改代码**。

## 文档地图

| 类型 | 路径 | 记录什么 | 更新时机 |
|------|------|----------|----------|
| 文档导航 | `docs/README.md` | 入口索引 | 新增文档类型时 |
| 产品需求（PRD） | `docs/product-requirements.md` | 当前有效范围、用户故事、验收 | 需求讨论后、变更确认后 |
| 产品设计（可选） | `docs/product-design.md` | 流程与交互原则 | 设计变更时 |
| 开发计划 | `docs/development-plan.md` | 里程碑、勾选状态 | 立项、排期变化、完成里程碑 |
| 技术规格 | `docs/development-guide.md` | 架构、接口、数据 | 改实现方案时 |
| 需求变更台账 | `docs/changes/` | 追加式变更单 | 每次范围/验收变化 |
| 架构/需求决策 | `docs/decisions/`（ADR） | 重大决策与原因 | 易反悔或分叉时 |
| 对外变更 | `CHANGELOG.md` | 版本维度变更 | 发版前 |
| 用户帮助 | `docs/help/` 或 \<外部 URL\> | 终端用户操作说明 | 功能对用户可见时 |
| 对内日志（可选） | `docs/journal/YYYY-MM.md` | 进展与卡点 | 随时 |

## 真源优先级（冲突时）

1. 已采纳的 ADR  
2. 当前 PRD（范围与验收）  
3. 技术规格（与产品冲突时以 PRD + ADR 为准）  
4. 开发计划（是否交付以计划 + 测试为准）

## 核心工作流

明确需求 → 更新 PRD/计划/变更或 ADR → 评审 → 实现 → 测试 → 更新计划勾选与 CHANGELOG → 更新用户帮助（若适用）→ 提交

```
