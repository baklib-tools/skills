# 模板与定义（用户故事、验收、变更、how-to）

可复制到各项目文档中，按团队习惯改名。

## 用户故事（简版）

```text
作为 <角色>
我希望 <能力>
以便 <业务价值 / 可观测结果>
```

建议为每条故事分配稳定 ID：**`US-042`**，并在 PRD、变更单、提交说明中引用同一 ID。

## 验收标准（Given / When / Then）

```markdown
**场景：<短标题>**
- **Given** <前置数据与配置>
- **When** <用户或系统动作>
- **Then** <可观察结果：界面/API/数据；尽量可自动化断言>
```

## 需求变更单（单篇追加）

```markdown
# CR-003 — <短标题>

- **日期**：YYYY-MM-DD
- **提出方**：
- **关联故事**：US-042（如有）
- **原条款摘要**：（或指向 PRD 章节）
- **新条款摘要**：
- **原因**：
- **对范围/工期/成本的影响**：（若未知写「待评估」）
- **确认状态**：未确认 / 已确认（谁、何时）
```

## 用户可见功能的 Definition of Done（建议）

- [ ] 验收标准（含 GWT）已通过或等价签字  
- [ ] `development-plan`（或等价计划）已勾选或更新  
- [ ] 若对终端用户可见：**帮助文档**已更新，或已建文档工单并在发版说明中有一句用户可读摘要  
- [ ] 对外行为变化已反映在 `CHANGELOG`（若项目要求）
