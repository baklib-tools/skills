# 限制与运行环境

## 过滤器严格模式

主题引擎对过滤器启用 **`strict_filters: true`**：使用**未注册**的过滤器名会导致错误。请仅使用 Liquid 内置过滤器与本技能 `filters.md` 中列出的自定义过滤器。

## 变量未定义

渲染时 **`strict_variables: false`**：访问不存在的变量通常得到空值，而不一定抛错；仍建议在逻辑中显式判断。

## 分页 `paginate_tag`

- **`per`**：每页条数，**默认 10**，**最大 50**（超过按上限处理）。
- 页码参数默认从请求参数 `page` 或 `_page` 读取（见分页指令实现）。

## 静态页与反向代理

- 静态页 URL 在默认路由下常带有 **`/s/`** 前缀；站点若配置在域名子路径下，还需 **`script_name`** 前缀。
- **正确做法**：用 `{{ statics['路径'] }}` 生成静态页完整 URL；站内页面链接用 `page.path`、`site.pages['路径']` 等由系统生成的路径。
- **避免**：在模板中写死 `/s/xxx` 或省略 `script_name` 导致子目录部署时 404。
- **必须拼接字面路径时**：使用 `{{ script_name }}/s/xxx` 等形式，使前缀随请求一致。

## 事件追踪 `track_event`

- 事件名字符串必须在主题 **`config/settings_schema.json`** 的 **`track_events`** 分组中预先声明；未声明的事件会被忽略。
- **预览模式**下不记录事件。

## 开发环境

- 开发环境下可能注册 **`debug`** 指令（仅供调试）；生产主题请勿依赖。

## 导航树规模

- `site.nav_tree` 内部对节点总量有上限（例如一万级）与缓存策略；超大数据量时注意性能，按需限制 `depth`。
