技能小测验：微信公众号 HTML 排版要点

baklib-tools/skills · examples 演示文稿
导读：本页对应技能 wechat-mp-html。读完三道题，你能复述：粘贴到公众号的正文根节点、样式写在哪里、复制按钮的标准实现。
01
判断题
复制到公众号图文正文时，应复制 #js_content 这一段的 outerHTML，而不是整页 HTML。对吗？（答案：对。标题与作者应在后台单独填写。）
02
选择题
正文样式应主要写在哪里？
A. 仅写在页面 <head> 里的 <style>，粘贴时一起过去
B. 写在每个元素的 inline style，不依赖正文里的 class
C. 写在微信公众号后台「自定义 CSS」
答案：B。公众号粘贴不会带上本页 <head> 样式，故组件须自带 style="…"。
动手：复制按钮
标准实现使用按钮 id #copy-to-weixin、提示 #copy-toast，以及 ClipboardItem 同时写入 text/html 与 text/plain。本页右下角即为该按钮，可在本地点按试复制。
03
简答
发布前，文内图片的 src 应如何处理？
参考答案
相对路径仅便于本地预览；正式发布须改为已在公众号素材库上传后的 https 图片地址，或在后台插入图片生成链接。本示例使用公开占位图仅作演示。
